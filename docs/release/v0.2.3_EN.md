# Release v0.2.3

**Release Date**: 2025-12-08

## Overview

v0.2.3 is a feature enhancement release. validategen adds the `@oneof_enum` annotation for enum type validation, a new golangcilint tool for IDE diagnostics integration, and the VSCode extension supports global validation on startup.

## New Features

### validategen: @oneof_enum Enum Type Validation

Added `@oneof_enum(EnumType)` annotation to validate that a field value is a valid value of the specified enum type. Unlike `@oneof`, `@oneof_enum` automatically uses the `EnumTypeEnums.Contains()` method generated by enumgen, avoiding the need to modify `@oneof` annotations when adding new enum values.

**Same package enum:**
```go
// enumgen:@enum(string)
type Role int

const (
    RoleAdmin Role = iota
    RoleUser
    RoleGuest
)

// validategen:@validate
type User struct {
    // validategen:@oneof_enum(Role)
    Role Role  // Automatically uses RoleEnums.Contains() for validation
}
```

**Cross-package enum (full import path):**
```go
// validategen:@validate
type Request struct {
    // validategen:@oneof_enum(github.com/myorg/pkg/types.Status)
    Status types.Status  // Automatically imports "github.com/myorg/pkg/types"
}
```

**Cross-package enum (with import alias):**
```go
// validategen:@validate
type Request struct {
    // validategen:@oneof_enum(mytypes:github.com/myorg/pkg/types.Status)
    Status mytypes.Status  // Uses the specified alias
}
```

**Advantage**: When adding new enum values, you only need to modify the enum definition, not the `@oneof_enum` annotation, following the single modification principle.

### golangcilint: IDE Diagnostics Integration Tool

Added `golangcilint` tool for IDE diagnostics integration. This tool can be used as a custom linter for golangci-lint, displaying devgen annotation errors and warnings in real-time in the editor.

### VSCode Extension: Global Validation on Startup

The VSCode extension now supports running global dry-run validation on startup, automatically detecting devgen annotation issues in all Go files in the workspace and displaying diagnostics in the Problems panel.

## Improvements

### Enhanced devgen config Command

The `devgen config` command now outputs more detailed help information, including:
- All available tools and their annotations
- Parameter types and allowed values
- Configuration file format description
- Output format options

## Upgrade Guide

### Upgrading from v0.2.2

Direct upgrade with no breaking changes:

```bash
go install github.com/tlipoca9/devgen/cmd/devgen@v0.2.3
```

If using `@oneof_enum` validation, re-run `devgen` to generate code.
