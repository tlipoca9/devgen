# Release v0.1.0

**Release Date**: 2025-12-07

## Overview

First release of devgen! A Go code generation toolkit that automatically generates boilerplate code through annotations, reducing repetitive manual coding.

## enumgen - Enum Code Generator

Automatically generates serialization, deserialization, and validation methods for Go enum types. Say goodbye to manually writing tedious `String()`, `MarshalJSON()` methods.

### Usage

1. Add annotations:

```go
// Status represents status
// enumgen:@enum(string, json, sql)
type Status int

const (
    StatusPending Status = iota + 1
    StatusActive
    // enumgen:@name(Cancelled)
    StatusCanceled  // Custom name
)
```

2. Run code generation:

```bash
enumgen ./...
```

### Features

| Annotation | Description |
|------------|-------------|
| `@enum(string)` | Generate `String()` method |
| `@enum(json)` | Generate `MarshalJSON()` / `UnmarshalJSON()` |
| `@enum(text)` | Generate `MarshalText()` / `UnmarshalText()` |
| `@enum(sql)` | Generate `Value()` / `Scan()` for database operations |
| `@name(Name)` | Custom string name for enum value |

### Generated Helper Methods

Automatically generates `{Type}Enums` helper variable providing:
- `IsValid()` - Check if enum value is valid
- `List()` - Return all valid enum values
- `Parse(s)` - Parse enum from string
- `Name(v)` - Get string name of enum value
- `Contains(v)` / `ContainsName(s)` - Check if value or name is valid

## validategen - Validation Code Generator

Automatically generates `Validate()` methods for Go structs. Declare validation rules through annotations without writing validation logic manually.

### Usage

1. Add annotations:

```go
// User model
// validategen:@validate
type User struct {
    // validategen:@required
    // validategen:@min(2)
    // validategen:@max(50)
    Name string

    // validategen:@required
    // validategen:@email
    Email string

    // validategen:@gte(0)
    // validategen:@lte(150)
    Age int

    // validategen:@oneof(admin, user, guest)
    Role string
}
```

2. Run code generation:

```bash
validategen ./...
```

### Validation Annotations

| Category | Annotation | Description |
|----------|------------|-------------|
| **Required** | `@required` | Field cannot be empty/zero value |
| **Range** | `@min(n)` `@max(n)` | Numeric range or length limit |
| | `@len(n)` | Exact length |
| | `@gt(n)` `@gte(n)` `@lt(n)` `@lte(n)` | Greater/less than comparisons |
| **Equality** | `@eq(v)` `@ne(v)` | Equal/not equal |
| | `@oneof(a, b, c)` | Must be one of specified values |
| **Format** | `@email` `@url` `@uuid` | Common format validation |
| | `@ip` `@ipv4` `@ipv6` | IP address validation |
| **Character** | `@alpha` `@alphanum` `@numeric` | Character type restrictions |
| **String** | `@contains(s)` `@excludes(s)` | Contains/excludes substring |
| | `@startswith(s)` `@endswith(s)` | Prefix/suffix match |
| **Regex** | `@regex(pattern)` | Custom regular expression |
| **Data Format** | `@format(json\|yaml\|toml\|csv)` | Validate data format |
| **Nested** | `@method(MethodName)` | Call nested struct's validation method |

### Advanced Features

- **postValidate Hook**: Define `postValidate(errs []string) error` method for custom validation logic
- **Multiple Rules**: A field can use multiple validation annotations simultaneously
- **Detailed Error Messages**: Error messages include field name and actual value for easy debugging

## Installation

### Install devgen

```bash
# Install devgen tool, supports all devgen tools (including enumgen, validategen)
go install github.com/tlipoca9/devgen/cmd/devgen@latest

# usage
devgen ./...
```

### Install Individual Tools

```bash
go install github.com/tlipoca9/devgen/cmd/enumgen@latest
go install github.com/tlipoca9/devgen/cmd/validategen@latest

# usage
enumgen ./...
validategen ./...
```
