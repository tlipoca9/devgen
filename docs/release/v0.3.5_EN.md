### validategen: @default Annotation Support

New `@default` annotation for setting default values on struct fields. When a field has a zero value, the default value is automatically applied.

#### Usage Example

```go
// validategen:@validate
type Config struct {
    // validategen:@default(localhost)
    Host string  // Default: "localhost"

    // validategen:@default(8080)
    Port int  // Default: 8080

    // validategen:@default(true)
    Enabled bool  // Default: true

    // validategen:@default(1.0)
    Version float64  // Default: 1.0
}
```

#### Generated Code

Generates a separate `SetDefaults()` method to call before validation:

```go
cfg := &Config{}
cfg.SetDefaults()  // Set default values
if err := cfg.Validate(); err != nil {
    // handle error
}
```

#### Supported Types

- `string` - Set default string value
- Numeric types (`int`, `int8`, `int16`, `int32`, `int64`, `uint`, `uint8`, `uint16`, `uint32`, `uint64`, `float32`, `float64`) - Set default numeric value
- `bool` - Set default boolean value (`true`/`false`)

### validategen: SetDefaults Test Generation

When using the `--include-tests` flag, test code for `SetDefaults()` methods is automatically generated:

```bash
devgen --include-tests ./...
```
