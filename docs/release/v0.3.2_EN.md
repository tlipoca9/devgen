# v0.3.2 Release Notes

## Bug Fixes

### validategen: Fix `@oneof_enum` support for string field types and cross-package imports

**Issues:**

1. `@oneof_enum` annotation previously only supported custom enum type fields, not `string` type fields
2. When enum types are defined in other packages, generated validation and test code was missing necessary import statements

**Fixes:**

1. **Support `@oneof_enum` on string type fields**

   You can now use `@oneof_enum` annotation on `string` type fields:

   ```go
   // validategen:@validate
   type Config struct {
       // validategen:@oneof_enum(Status)
       StatusStr string  // Validates value must be a valid Status enum string
   }
   ```

2. **Support cross-package enum types**

   When referencing enum types from other packages, generated code automatically adds correct imports:

   ```go
   import "myapp/common"

   // validategen:@validate
   type Order struct {
       // validategen:@oneof_enum(common.Priority)
       Priority string  // References Priority enum from common package
   }
   ```

3. **Test code cross-package import fix**

   Generated `*_validate_test.go` files now correctly import cross-package types.

**New examples:**

- `cmd/validategen/examples/common/` - Cross-package enum type examples
  - `types.go` - Defines `Status` enum
  - `common_enum.go` - Generated enum helper methods
